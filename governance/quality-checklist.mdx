---
title: "Checklist de qualidade"
description: "Critérios para revisar páginas, projetos filhos e agentes do CapyUniverse."
---

Use este checklist antes de publicar qualquer página ou módulo relevante.

## Conteúdo

- [ ] A página explica o objetivo em até 3 linhas.
- [ ] O público-alvo está claro.
- [ ] O status está claro: ideia, protótipo, beta ou produto.
- [ ] Funcionalidades reais estão separadas de roadmap.
- [ ] Riscos e limitações estão explícitos.
- [ ] Não há promessa exagerada.

## Técnica

- [ ] Stack documentada.
- [ ] Como rodar documentado.
- [ ] Dependências externas listadas.
- [ ] Variáveis de ambiente mencionadas sem expor valores.
- [ ] Módulos principais descritos.
- [ ] Próximos passos técnicos registrados.

## UX e produto

- [ ] O fluxo principal do usuário está claro.
- [ ] O problema resolvido está explícito.
- [ ] Existe próximo passo recomendado.
- [ ] Há critério para virar produto ou continuar experimento.

## Documentação

- [ ] Página existe no `docs.json` se for parte oficial.
- [ ] Links externos funcionam.
- [ ] Não existe conteúdo do template antigo.
- [ ] Nomes e termos seguem o glossário.
- [ ] O texto está em português claro.

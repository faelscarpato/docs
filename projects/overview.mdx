Os projetos filhos são produtos, utilitários, bases de conhecimento ou plataformas conectadas ao CapyUniverse.

## Mapa atual

| Projeto | Papel no universo | Status documental |
| --- | --- | --- |
| CapyUniverse Core | Plataforma-mãe e hub local | Principal |
| ValeRepor | Produto comercial para supermercados | Prioridade alta |
| Bicou | Marketplace/protótipo de bicos rápidos | Prioridade média/alta |
| CapyMind | Knowledge pack e memória documental | Prioridade alta |
| BuskaCEP | Utilitário cívico/local para CEPs de Pedreira-SP | Prioridade média |
| Sentinela Pedreira | Inteligência cívica municipal | Prioridade média/alta |

## Critério de classificação

| Tipo | Quando usar |
| --- | --- |
| Plataforma | Centraliza outros projetos ou módulos |
| Produto comercial | Tem cliente, dor e proposta vendável |
| Utilitário público | Resolve problema local ou comunitário |
| Knowledge base | Organiza conhecimento para humanos e IA |
| Laboratório | Testa tecnologia, UX ou automação |

## Prioridade recomendada

1. **CapyUniverse Core:** precisa ser a base conceitual e técnica.
2. **CapyMind:** ajuda a organizar conhecimento, contexto e governança.
3. **ValeRepor:** primeiro produto comercial claro.
4. **Bicou:** produto estratégico, mas com riscos jurídicos/operacionais maiores.
5. **Sentinela Pedreira:** projeto público com impacto, exige curadoria e responsabilidade.
6. **BuskaCEP:** utilitário prático e bom exemplo de produto local.

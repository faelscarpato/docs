## Resumo

CapyMind é uma base documental organizada para servir como contexto confiável para humanos, automações e IAs.

## Papel dentro do CapyUniverse

CapyMind deve funcionar como a **memória estruturada** do ecossistema: catálogo de projetos, instruções para agentes, referências, políticas, metadados, índices e documentação canônica.

## Objetivo

- Centralizar conhecimento útil do ecossistema Capy.
- Permitir recuperação confiável por IA.
- Reduzir ambiguidade, duplicação e drift documental.
- Aplicar padrões de segurança, citabilidade e manutenção.

## Escopo adequado

| Entra no CapyMind | Não deve entrar sem curadoria |
| --- | --- |
| Guias | Backups brutos |
| Runbooks | Arquivos temporários |
| ADRs | Exports pesados |
| Catálogo de projetos | Cópias duplicadas de repositórios |
| Políticas de IA | Dados sem classificação |
| Schemas e exemplos | Conteúdo sensível sem regra |

## Relação com esta documentação

- **CapyUniverse Docs:** vitrine pública/organizada do ecossistema.
- **CapyMind:** base de conhecimento e contexto para agentes.

## Evolução recomendada

1. Manter catálogo de projetos atualizado.
2. Criar metadados por projeto.
3. Criar indexação para IA.
4. Criar política de fontes e citações.
5. Integrar com agentes do CapyUniverse.

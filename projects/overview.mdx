A regra central é simples: **não completar lacunas por imaginação**. Se o repositório, README, CSV ou documentação fornecida não provar uma informação, ela deve aparecer como **INFORMAÇÃO NÃO FORNECIDA**.

## Critério de classificação

A classificação por categoria é editorial e serve para navegação. Ela foi derivada de nome, descrição, linguagem e acesso informados no CSV.

| Tipo | Quando usar |
| --- | --- |
| Plataforma | Centraliza outros projetos, módulos ou agentes |
| Produto comercial | Tem cliente, dor e proposta vendável |
| Utilitário público | Resolve problema local ou comunitário |
| Knowledge base | Organiza conhecimento para humanos e IA |
| Laboratório | Testa tecnologia, UX ou automação |
| Backlog privado | Existe no CSV, mas não deve ter detalhes inventados |

<Warning>
  Repositórios privados ou sem descrição suficiente não devem receber funcionalidades, stack detalhada ou roadmap por inferência. A página deve indicar o que falta consultar.
</Warning>

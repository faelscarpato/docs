---
title: "Padrão de uma CapyTool"
description: "Modelo para criar novas ferramentas dentro do CapyUniverse."
---

Uma **CapyTool** é uma ferramenta modular com entrada, processamento e saída bem definidos.

## Estrutura mínima

Toda CapyTool deve ter:

| Campo | Descrição |
| --- | --- |
| Nome | Nome curto e memorável |
| Objetivo | Problema que resolve |
| Usuário | Quem usa a ferramenta |
| Entrada | Texto, arquivo, imagem, dados ou formulário |
| Processamento | IA, regra local, API ou pipeline |
| Saída | Texto, imagem, relatório, arquivo ou ação |
| Dependências | Bibliotecas, APIs, chaves e dados |
| Riscos | Segurança, privacidade, custo ou erro de IA |
| Status | Ideia, protótipo, beta, produto |

## Template rápido

```md
# CapyTool: [nome]

## Objetivo

## Usuário-alvo

## Entrada

## Saída

## Fluxo

## Dependências técnicas

## Limitações atuais

## Roadmap
```

## Critérios para virar produto filho

Uma CapyTool pode virar projeto filho quando tiver:

- dor clara;
- público definido;
- fluxo repetível;
- valor comercial ou operacional;
- documentação mínima;
- roadmap próprio;
- separação técnica suficiente para evoluir sem quebrar o Core.

<Note>
  Nem toda ferramenta precisa virar produto. Algumas ferramentas existem apenas para acelerar o desenvolvimento dos outros projetos.
</Note>

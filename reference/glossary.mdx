## Termos principais

| Termo | Significado |
| --- | --- |
| CapyUniverse | Plataforma-mãe do ecossistema de ferramentas, agentes e projetos |
| CapyTool | Ferramenta modular com função específica dentro do universo |
| Agente IA | Função especializada baseada em prompt, regras e contexto |
| Projeto filho | Produto ou sistema conectado ao ecossistema principal |
| Core | Base central do CapyUniverse |
| Local-first | Estratégia que prioriza funcionamento local antes de depender de backend |
| Knowledge pack | Base documental estruturada para humanos e agentes |
| Prompt mestre | Instrução principal de um agente ou fluxo IA |
| Roadmap aberto | Plano evolutivo sujeito a mudança conforme aprendizado e uso real |
| Docs-as-code | Documentação mantida em arquivos versionados no Git |

## Status de projeto

| Status | Uso |
| --- | --- |
| Ideia | Ainda não há fluxo implementado |
| Protótipo | Fluxo existe, mas não é produto pronto |
| Beta | Pode ser testado por usuários selecionados |
| Produto | Tem proposta clara, fluxo estável e documentação operacional |
| Arquivado | Não está em evolução ativa |

## Classificação de módulos

| Classificação | Exemplo |
| --- | --- |
| Plataforma | CapyUniverse Core |
| Produto comercial | ValeRepor, Bicou |
| Produto público/local | BuskaCEP, Sentinela Pedreira |
| Base de conhecimento | CapyMind |
| Ferramenta interna | CapyOCR, CapyPDF, CapyPlanner |

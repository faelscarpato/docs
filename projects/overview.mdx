## Critério de classificação

| Tipo | Quando usar |
| --- | --- |
| Plataforma | Centraliza outros projetos, módulos ou agentes |
| Produto comercial | Tem cliente, dor e proposta vendável |
| Utilitário público | Resolve problema local ou comunitário |
| Knowledge base | Organiza conhecimento para humanos e IA |
| Laboratório | Testa tecnologia, UX ou automação |

## Regra documental

Cada página de projeto deve responder:

- O que é o projeto na prática.
- Qual papel ele ocupa dentro do CapyUniverse.
- O que existe hoje, com base no repositório público.
- Qual stack foi verificada.
- Como rodar, quando o repositório informa.
- Quais limitações e riscos estão explícitos.
- O que ainda não foi fornecido.

<Note>
  Quando uma informação não aparece nas fontes públicas consultadas, a página deve registrar **INFORMAÇÃO NÃO FORNECIDA** em vez de completar a lacuna por inferência.
</Note>

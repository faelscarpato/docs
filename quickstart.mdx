Este guia mostra como pensar o CapyUniverse como **base local dos seus projetos**.

<Steps>
  <Step title="Entenda a função do CapyUniverse">
    O CapyUniverse não é apenas um site com várias páginas. Ele é o **hub central** onde cada página, ferramenta ou agente representa uma função específica dentro de um universo maior.
  </Step>
  <Step title="Rode o projeto localmente">
    Use o fluxo padrão do projeto principal:

    ```bash
    npm install
    npm run dev
    ```

    Depois configure as variáveis necessárias no `.env`, como chaves de provedores IA quando forem usadas.
  </Step>
  <Step title="Escolha um tipo de módulo">
    Cada nova ideia deve entrar em uma das categorias:

    - **CapyTool:** ferramenta funcional.
    - **Agente:** IA com papel especializado.
    - **Produto filho:** projeto com público, problema e roadmap próprios.
    - **Experimento:** protótipo em validação.
  </Step>
  <Step title="Documente antes de crescer">
    Antes de criar muitas páginas, registre objetivo, entrada, saída, dependências, riscos e critérios de aceite do módulo.
  </Step>
</Steps>

## Estrutura mental do universo

```txt
CapyUniverse
├── Plataforma Core
│   ├── Interface principal
│   ├── Navegação
│   ├── Serviços IA
│   └── Estado local
├── CapyTools
│   ├── CapyOCR
│   ├── CapyPDF
│   ├── CapyPlanner
│   └── Novas ferramentas
├── Agentes
│   ├── Engenharia
│   ├── Infográficos
│   ├── Renderização
│   └── Documentação
└── Projetos filhos
    ├── ValeRepor
    ├── Bicou
    ├── CapyMind
    ├── BuskaCEP
    └── Sentinela Pedreira
```

## Decisão prática

<Tip>
  Sempre que surgir uma nova ideia, pergunte: isso é uma ferramenta, um agente, um produto filho ou apenas um experimento? Essa decisão evita bagunça no ecossistema.
</Tip>
